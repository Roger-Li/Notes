Manipulate Bayes Rule

We have the bayes rule as follows:

p(A∣B)=p(B∣A)P(A)P(B)p(A|B)=\dfrac{p(B|A)P(A)}{P(B)} p(A∣B)=P(B)p(B∣A)P(A)​

If we don't have p(B)p(B)p(B) directly, we can calculate it from the numerator, i.e.

P(B)=∫AP(A,B)=∫Ap(B∣A)P(A)P(B) = \int_A P(A,B) = \int_Ap(B|A)P(A) P(B)=∫A​P(A,B)=∫A​p(B∣A)P(A)

We can think of the bottom term as a "normalization constant" so that the distribution sums up to 1

p(A∣B)=p(B∣A)P(A)∫Ap(B∣A)P(A)=p(B∣A)P(A)Z∝p(B∣A)P(A)p(A|B)=\dfrac{p(B|A)P(A)}{\int_Ap(B|A)P(A)} = \dfrac{p(B|A)P(A)}{Z} \propto p(B|A)P(A) p(A∣B)=∫A​p(B∣A)P(A)p(B∣A)P(A)​=Zp(B∣A)P(A)​∝p(B∣A)P(A)

as ZZZ is independent of AAA. This could be useful if we want to optimize over AAA, i.e.,

argmaxAP(A∣B)=argmaxAP(B∣A)P(A)\text{argmax}_A P(A|B) = \text{argmax}_A P(B|A) P(A) argmaxA​P(A∣B)=argmaxA​P(B∣A)P(A)

Test the Mean Difference Between Two Groups

t Test

Assuming all data are iidiidiid Gaussian, we have X1={x1(1),...,x1(N1)},X2={x2(1),...,x2(N2)}X_1 = \{x_1^{(1)}, ..., x_1^{(N_1)}\}, X_2 = \{x_2^{(1)}, ..., x_2^{(N_2)}\}X1​={x1(1)​,...,x1(N1​)​},X2​={x2(1)​,...,x2(N2​)​}, and we want to test whether or not group 1 is different than group 1 on average

  • Null hypothesis H0:μ1−μ2≠0H_0: \mu_1 - \mu_2 \neq 0H0​:μ1​−μ2​​=0
  • The test statistic t=(X1ˉ−X2ˉ)−(μ1−μ2)S12N1+S22N2t=\dfrac{(\bar{X_1}-\bar{X_2})-(\mu_1 - \mu_2)}{\sqrt{\frac{S_1^2}{N_1}+\frac{S_2^2}{N_2}}}t=N1​S12​​+N2​S22​​​(X1​ˉ​−X2​ˉ​)−(μ1​−μ2​)​
  • If the null hypothesis is true, t∼TN1+N2−2t\sim T_{N_1+N_2-2}t∼TN1​+N2​−2​
  • The C.I. for the difference in means μ1−μ2\mu_1 - \mu_2μ1​−μ2​ is given by (X1ˉ−X2ˉ)±t∗S12N1+S22N2(\bar{X_1}-\bar{X_2}) \pm t^*\sqrt{\frac{S_1^2}{N_1}+\frac{S_2^2}{N_2}}(X1​ˉ​−X2​ˉ​)±t∗N1​S12​​+N2​S22​​​ where t∗t^*t∗ is the 1−α/21-\alpha/21−α/2 (e.g., 0.9750.9750.975) critical value.

Pooled t Procedure

Note that this is to assume that the two group has different variance, if we know that the two groups have the same variance (standard deviation), we use the pooled t procedure where we calculate the pooled sample variance $$S_p^2=\frac{(N_1-1)S_1^2+(N_2-1)S_2^2}{N_1+N_2-2}$$, and the test statistic becomes $$t=\dfrac{(\bar{X_1}-\bar{X_2})-(\mu_1 - \mu_2)}{S_p\sqrt{\frac{1}{N_1}+\frac{1}{N_2}}}$$

P-Value

  • Definition: The probability of obtaining a result equal to or 'more extreme' than what was actually observed, when the null hypothesis is true.
  • Since we use a significance level α\alphaα, we reject the null hypothesis if p-value ≤α\leq \alpha≤α

Distribution-free Tests

  • If we don't want to assume Gaussian or any distribution, we could apply non-parametric tests such as
    • Kolmogorov-Smirnov test
    • Kruskal-Wallis test
    • mann-Whitney U test
  • Less assumptions →\rightarrow→ less power →\rightarrow→ need "more extreme" observed difference for statistically significant p-value.

A/B Testing Pitfalls

  • Should not stop experiment early when it reaches significance
  • Choosing Sample Size: general rule of thumb

N=16σ2δ2N=16\frac{\sigma^2}{\delta^2} N=16δ2σ2​

where σ2\sigma^2σ2 is the variance of data, δ\deltaδ is the minimum difference you want to detect

  • An example why A/B testing in Frequentist setting could be undesirable New drug or landing page is working well, but you cannot stop the test to launch it early because it increases the chance of FPs. Remember, p-value is the probability that observations show difference even if they are actually the same (i.e. null hypothesis is true), and it can also pass below the threshold and go back up over time.

Sec 4: Bayesian A/B testing

Epsilon-Greedy algorithm

  • one way to solve the exploration-exploitation dilemma, also used in reinforcement learning
  • Choose a small number ϵ∈(0,1)\epsilon \in (0,1)ϵ∈(0,1) and let it be the probability of exploration, i.e. choose at random. With probability of 1−ϵ1-\epsilon1−ϵ we exploit , i.e., choose the current-best arm (multi-armed bandit).
  • Note that the rate of exploration and exploitation are fixed. In the long run, the probability of choosing the best arm is 1−ϵ/21-\epsilon/21−ϵ/2. In the case of CTR, if reward is 111 for click and 000 for no click, the loss is ϵN/2\epsilon N/2ϵN/2, which is O(N)O(N)O(N).

Upper Confidence Bound (UCB1) Algorithm

Chernoff-Hoeffding's inequality

Let X1,...,XNX_1,...,X_NX1​,...,XN​ be independent r.v.s in the range [0,1][0,1][0,1] with E[Xi]=μE[X_i]=\muE[Xi​]=μ, then for ϵ>0\epsilon >0ϵ>0, we have $$p(\hat{\mu }\geq \mu + \epsilon)\leq \exp(-2\epsilon^2N)$$ The opposite is also true, so

p(∣μ^−μ∣≤ϵ)>1−2exp⁡(−2ϵ2N)p(|\hat{\mu}-\mu|\leq \epsilon)>1-2\exp(-2\epsilon^2N) p(∣μ^​−μ∣≤ϵ)>1−2exp(−2ϵ2N)

UCB1 algorithm and properties

  • For each arm jjj, choose with probability $$\epsilon_j=\sqrt{\frac{2\ln N}{N_j}}$$ Where N=N=N= # of total games played so far, Nj=N_j=Nj​= total # times arm jjj is played.
  • For each pick, choose $$j^*=\text{argmax}(\hat{\mu}_j + \sqrt{\frac{2\ln N}{N_j}})$$
  • First term obviously represents exploitation. The second term depends on both NNN and NjN_jNj​, if NjN_jNj​ is low, we are not so confident about the estimate of jjj, thus explore it more.
  • Since ln⁡NN→0\frac{\ln N}{N}\rightarrow 0NlnN​→0, we are approaching the true CTR estimate asymptotically.
  • In the long run, the loss of UCB1 is O(ln⁡N)O(\ln N)O(lnN), which is much less than the Epsilon-Greedy algorithm.

Regret (expected loss)

The regret of a policy is defined as $$\mu^*N-\sum_{j=1}^{K}\mu_j\cdot E[T_j(N)]$$ where μj\mu_jμj​ is the expected reward of arm jjj, μ∗\mu^*μ∗ is the expected reward of the optimal arm, TjT_jTj​ is the # of times arm jjj was played.

It represented the expected loss after NNN plays due to the fact that the policy does not always play the optimal arm.

Bayesian A/B Testing with Thompson Sampling

  • Thompson Sampling "is a heuristic for choosing actions that addresses the exploration-exploitation dilemma in the multi-armed bandit problem. It consists in choosing the action that maximizes the expected reward with respect to a randomly drawn belief."
  • Thompson sampling idea: pick action jjj with probability that increases with the probability (under posterior distributions) that jjj is optimal
while True:
    Draw a random sample from all bandits' current Beta(alpha_j, beta_j)
    j* = bandit that gives us maximum sample
    x = play bandit j*
    alpha_j* = alpha_j* + x
    beta_j* = beta_j* + 1 - x
  • See Python implementation by lazyprogrammer.
  • Bayesian method automatically converges to the maximum CTR, as demoed here. Intuitively, Thompson Sampling converges at exploiting the arm with the highest reward.
